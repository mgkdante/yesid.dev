● Footer

3. Form Terminal (Right)

yesid@mtl ~ /contact
~ $ yesid --contact
→ Opening contact form...
name:

~ $
Title bar: yesid@mtl ~ /contact
Command: $ yesid --contact + "Opening contact form..."
Fields: name, email, message — mono font, #0D0D0D bg
Focus: Input border glows orange
Submit: $ send --message → button
Validation: Inline ✗ under each field, red border, error count summary

4. Success State

yesid@mtl ~ /contact
~ $ send --message
→ Validating fields...
✓ name: OK
✓ email: OK
✓ message: OK
→ Sending message...
✓ Message sent successfully!
→ I'll get back to you within 24h
→ Meanwhile, check out my work or blog
~ $ reset --form
Trigger: On valid submit
Animation: Form fields fade out, typed sequence appears line by line (GSAP)
Checks: ✓ per field, then send + success
Links: "work" → /services, "blog" → /blog
Reset: $ reset --form restores the form
Note: No real backend yet (placeholder). Cloud slice adds real submission.

5. Mobile Layout

● Single column — terminals stack vertically
● Info terminal first (context), form terminal second (action)
● Both terminals full width
● Same interactions — hover becomes tap on links

6. Data Layer

● All text in contactContent object using LocalizedString
● Terminal commands, labels, placeholders, error messages — all data-driven
● Social links from siteMeta.links (already exists)
● Availability text reused from aboutPageContent.cta.availability
● Zero hardcoded strings in template

7. Animations

● Page entrance: use:reveal on both terminals (stagger left → right)
● Info hover: CSS transition — bg + text color on CONNECT lines
● Form focus: CSS transition — border color glow orange
● Submit success: GSAP typed sequence — lines appear one by one (~150ms stagger)
● Blinking cursor: CSS animate-pulse on both terminals

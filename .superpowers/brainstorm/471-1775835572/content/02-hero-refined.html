Architecture: Live + Mock Fallback

Live mode: Hero component fetches from your API endpoint (Oracle VM / Postgres). Metrics and SQL panel update every 30s matching your pipeline cadence.

Mock fallback: If API returns error or times out (>3s), component seamlessly falls back to hardcoded mock data that looks identical. No loading spinners, no error states — the visitor never knows.

Data sources from your Gold layer:

  • kpi_active_vehicles_latest → VEHICLES TRACKED card
  • kpi_avg_trip_delay_latest → AVG DELAY card
  • kpi_routes_with_live_vehicles_latest → ROUTES LIVE card
  • latest_trip_delay_snapshot → SQL panel results (top routes by vehicle count)

Phase 1 (this slice): Mock data, API-ready interface. Phase 2 (future): Wire to live Oracle VM API.

Services Index Page — Layout Direction

How should /services present yesid.'s 6 capabilities? Pick the layout that feels right for a consultative services page.

A

Metro Station Grid

Lean into the metro metaphor. Each service is a "station card" with the station number, SVG icon, title, description, and project count. 2-column grid on desktop, single on mobile. Hero banner at top: "What I Build" with a one-liner about yesid.'s capabilities. Cards link to /services/[id].

Pros

  • Ties into site's metro identity
  • Compact, scannable
  • Station numbers create visual order

Cons

  • Cards can feel samey with 6 items
  • Less room for SVG illustrations
B

Full-Width Sections (Stacked)

Each service gets a full-width horizontal section: large SVG on one side, title + description + "X projects" link on the other. Alternating left/right layout. Hero at top. Scrolls vertically through all 6. More breathing room for the SVG illustrations.

C

Hybrid: Hero + Featured Grid + Expandable

Hero section with tagline. Below it, 2-3 "core" services shown larger (cards with SVG, description, stack preview). Then the remaining services in a compact grid below. Click any → detail page. Gives visual hierarchy: core competencies stand out, supporting services are present but secondary.

Design Decisions
Reuses WorkDetailPage pattern: Collapsible sections with the same chevron toggle, same spacing, same reveal animations. Consistent UX across /work and /services detail pages.

Content sections from data: "How This Helps You" (valueProposition), "Typical Deliverables" (deliverables[]), plus custom ServiceSection[] entries — all collapsible, all LocalizedString.

Related projects at bottom: Full project cards on desktop (3-column grid with thumbnail, title, one-liner, stack). Compact list on mobile. All link to /work/[slug].

Prev/Next nav: Navigate between services without going back to the index. Station-order navigation.

No ToC on detail page: Services have fewer sections than work pages — a ToC would be overkill. The collapsible sections are enough.

Motion: use:reveal on each section. SplitText on the title. DrawSVGPlugin on the SVG (same as index). stagger on deliverables list and project cards.

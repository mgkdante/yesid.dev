3. Architecture Summary

/work (Listing)
  • Card grid layout (2-col desktop, 1-col mobile)
  • Tag filter pills with FLIP animation
  • Service filter pills (filter by service)
  • Cards show: title, oneLiner, service SVGs, tags
  • Click → /work/[slug]
  • 404 for invalid slugs
/work/[slug] (Detail)
  • Two-column: content left, sticky sidebar right
  • Hero: title + morph icon (D) + data-flow diagram (B)
  • Curated sections (scroll-revealed)
  • README section (last, labeled, from GitHub URL)
  • Sidebar: tech stack, links, service badges
  • Mobile: sidebar collapses above content
Data Model Changes
  • Project gets relatedServices: string[]
  • Project gets readmeUrl?: string
  • Service gets svg: string (SVG filename)
  • Service gets visible?: boolean (default true)
  • 2 new services added (Internal Tooling, Web Dev)
  • 4 new seed projects (placeholder content)
Animation System
  • 6 service SVGs (DrawSVG entrance + MorphSVG hover)
  • Data-flow diagram component (auto from stack[])
  • FLIP animation on grid filter
  • Scroll-reveal on sections (existing use:reveal)
  • Tag stagger animation (existing TagList)
  • Hero morph icon on page load

A. Card Grid + FLIP Filter

Visual grid with thumbnail placeholders. Tag pills filter with FLIP animation (cards rearrange smoothly). Great for visual portfolios. Matches the original PLAN.md spec.

Pros

  • Visual impact — thumbnail area makes each project pop
  • FLIP animation looks premium when filtering
  • Scales well to 6-12+ projects

Cons

  • Thumbnails are empty until you add real screenshots
  • More complex animation work (FLIP)

B. Row List (Blog-style)

Compact rows with small icon, title, excerpt, and tag pills. Similar to the blog listing pattern you already have. Simple filter with fade transitions.

Pros

  • Consistent with blog — same UX language
  • Works great even without thumbnails/screenshots
  • Faster to build (reuse blog patterns)

Cons

  • Less visual impact — no FLIP wow-factor
  • May feel "samey" with blog

C. Hybrid — Featured Hero + Grid

Featured project gets a large hero card at top. Remaining projects in a smaller grid below. Tag filter applies to the grid only. Featured stays pinned.

Pros

  • Highlights your best work immediately
  • Grid still gets FLIP animation
  • Hierarchy guides the eye

Cons

  • Requires at least one "featured" project worth highlighting
  • Most complex layout to build

My recommendation: A (Card Grid) — it differentiates /work from /blog visually, FLIP animation adds premium feel, and thumbnail placeholders are fine for now since you'll add real content later.

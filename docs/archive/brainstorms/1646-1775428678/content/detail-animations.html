Work Detail — Animation Concepts

What motion should the /work/[slug] page have? (Can combine multiple)

A

Per-Project SVG Illustration

Each project gets a custom SVG in the hero area (like blog posts get SVG icons). GSAP draws it on page load — lines trace, shapes fill, elements stagger in. For placeholder projects, use geometric fallbacks (hexagons, circuit-board patterns).

Fully revealed

Effort: Medium — reuse BlogSvgIcon pattern. Need 1 SVG per project (or geometric fallbacks).

B

Animated Data-Flow Diagram

Each project gets an auto-generated "pipeline schematic" based on its tech stack. Nodes represent technologies, animated connection lines show data flow between them. Very on-brand for a data engineer. GSAP staggers nodes in, then animates connecting paths with DrawSVG.

Effort: Higher — need a component that generates SVG from stack[] array. But auto-generates for every project with zero manual SVG work.

C

Scroll-Reveal Sections + Stagger Tags

No custom SVG — instead, lean into scroll motion. Each section card slides/fades in on scroll (use:reveal). Tech stack tags stagger in one-by-one on page load. Sidebar elements animate independently. Section headers get a subtle DrawSVG underline accent.

D

Hero Morph Icon (like Blog)

Each project gets a small geometric SVG icon (like blog posts). In the hero header, the icon plays a GSAP MorphSVG animation on load — morphing between shapes that represent the tech stack. Reuses the exact pattern from BlogSvgIcon.

Multi-select! These can combine. My recommendation: B + C — the data-flow diagram is uniquely on-brand for you as a data engineer, and scroll reveals are basically free since you already have the actions.

EDGE DETAILS — WHAT'S NEW

LEFT EDGE — STATUS READOUTS

Vertical text: "SEC—02" (section number), "MANIFESTO" (section name), "MTL—QC" (location). Active dot (glowing orange) + inactive dots (outline only). Like a control panel sidebar. Rotated via writing-mode.

RIGHT EDGE — COORDINATES

Montreal lat/long coordinates, node reference (berri-uqam callback), active status. Technical data aesthetic — like a network monitoring dashboard. All at ~15-25% opacity so it doesn't compete.

TOP EDGE — MEASUREMENT TICKS

Evenly spaced tick marks with pixel values (aligned to the 80px circuit grid). Like a ruler or technical drawing. Signals precision and engineering mindset.

BOTTOM EDGE — STATUS BAR

Pulsing dot + "CONNECTED" + URL + version + scroll hint. Like a terminal status bar or IDE footer. The pulsing dot is the only animation besides the cursor blink.

STRIPES — BOLDER

TL/BR at 18% opacity (was 10%), TR/BL at 9% (was 5%). Stripe width increased to 12px bands. Visible as a real design element now, not just a ghost.

GRID — UNIFORM

Back to uniform circuit grid from v2. No fade mask. Edge-to-edge coverage. Fills ALL margins with visual texture.

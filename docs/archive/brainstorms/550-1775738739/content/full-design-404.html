2. Animated Elements

Blinking construction lights: Two amber lights on the barriers alternate blinking (CSS animation, 1.2s interval). One on, one dim. Subtle — not distracting.
Station sign entrance: Fades in + slight scale-up from 0.9 → 1.0 on page load. 300ms ease-out.
Text stagger: "ROUTE NOT FOUND" types in letter by letter (monospace typewriter). Title and description fade in after. Route suggestion buttons stagger in 80ms apart.
Terminal line: The $ route --status 404 line types in character by character at the end. Cursor blink after completion.
Reduced motion: All animations collapse to simple opacity fade-in. Blinking lights stay static (one on, one dim).

3. Technical Details

Route: SvelteKit +error.svelte at root level. Catches all unmatched routes.
Illustration: Single SVG component — ConstructionScene.svelte. Scaffolding, barriers, cones, lights, station sign all inline SVG for crisp rendering at any size.
Copy: All text through LocalizedString (i18n-ready). "This station is under construction" / "Cette station est en construction" / "Esta estación está en construcción".
Responsive: SVG scene scales down on mobile. Route suggestions stack vertically below 480px. Full-bleed hazard stripes at all breakpoints.
Nav: The pill nav renders on the 404 page — users always have navigation available.

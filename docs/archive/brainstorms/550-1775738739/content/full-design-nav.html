3. Scroll Behavior

At rest (top of page): Pill floats with full opacity. Progress rail visible.
On scroll down: Pill stays fixed. Slight opacity increase on background for contrast against page content scrolling behind it. Progress rail segments fill as you scroll through sections.
No hide-on-scroll. The pill is small enough that it never feels intrusive. Always accessible.
Active link updates based on current route (SvelteKit $page.url.pathname).

Animation: Overlay scales up from pill position (transform-origin: pill center). ~250ms ease-out. Links stagger in 40ms apart along the metro line. Close = reverse collapse into pill. Fast, mechanical.
Metro line: Vertical line with stop dots. Current page = filled orange dot with glow. Others = hollow rings.
Subtitles: Monospace micro-descriptions under each link. Adds personality without clutter.
Pill persists: The pill stays at top, but the menu icon morphs to ✕.
Background: Solid dark with subtle radial orange gradient from top center.

A) Full Terminal — The Form IS a Terminal

One large terminal window dominates the page. Form fields styled as terminal prompts (name:, email:, message:). Input fields look like terminal input. Submit = send --message →. Social links below as plain text paths. Maximum cohesion with the CTA widget.

B) Split — Terminal Info + Clean Form

Left column: terminal window with status info + social links (same aesthetic as CTA). Right column: clean card with standard form fields. Terminal sets the mood but form stays conventional for usability. This is what the slice spec originally described.

C) Terminal Chrome Wraps Everything

One big terminal window with title bar, but inside it's a two-column layout: info/socials left, form right. Form inputs use mono font and darker backgrounds but are still real inputs. The terminal is the frame, not the UX. Best of both worlds — immersive but functional.

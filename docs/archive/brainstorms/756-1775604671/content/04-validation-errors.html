Contact — Validation Errors

User hits send --message → with missing or invalid fields. How does the terminal communicate errors?

yesid@mtl ~ /contact
~ $ yesid --contact
→ Opening contact form...
name:
✗ required — name cannot be empty
email:
not-an-email
✗ invalid — enter a valid email address
message:
I need help with my database...
✗ 2 errors — fix and retry
~ $ send --message →

A) Inline Under Each Field

Red ✗ message below each invalid field, border turns red. Valid fields get green border. Error count summary at bottom: ✗ 2 errors — fix and retry. Form stays visible. Most usable — user sees exactly what to fix.

yesid@mtl ~ /contact
~ $ send --message
→ Validating fields...
✗ name: required
✗ email: invalid format
✓ message: OK
ERROR: 2 fields need attention
→ Fix the fields above and retry
~

B) Terminal Validation Output

On submit, validation runs as terminal output below the form (same typed sequence as success). ✗ name: required, ✓ message: OK. Fields highlighted above. Scroll back up to fix. Most "terminal-authentic" but requires scrolling.

A is more usable (errors next to fields). B is more terminal-authentic (errors as command output). Pick one or mix.

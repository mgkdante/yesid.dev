A — Metro Stacked Cards

Each project is a full-width card on the metro line. Gradient hero banner per card with project title. Cards stack vertically — scroll to explore. Station numbers anchor each project. Immersive but scannable.

Pros

  • Each project gets a moment
  • Strong metro wayfinding
  • Gradient banners = visual variety

Cons

  • Long page with many projects
  • Less overview at a glance

B — Spotlight Grid

Grid of cards, but hover spotlights one: border glows, others dim to 30% opacity, excerpt expands. Like La Playa's award-winning pattern. Filters above. Interactive discovery.

Pros

  • Hover interaction = wow factor

Cons

  • No metro line (grid breaks linearity)
  • Less storytelling, more browsing

C — Metro Split Cards

Metro line + split-layout cards: text left, visual right (DataFlowDiagram or gradient). Each card is a mini story preview. Hover reveals CTA. Storytelling + transit + visual.

Pros

  • Metro line + visual richness
  • Split layout = story teaser
  • DataFlowDiagram reused inline

Cons

  • Cards are wider — 1 column only
  • Needs responsive split→stack

💡 My recommendation: C — the split card gives each project a mini case-study preview right on the listing. Metro line keeps the transit identity. DataFlowDiagram reuse means you're showing technical depth before the click. And it mirrors the blog's timeline approach — consistent system across both sections.

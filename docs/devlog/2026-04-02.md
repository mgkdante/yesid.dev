### Work Done

- [x] Create `src/lib/data/types.ts`
  - Files changed: `src/lib/data/types.ts`
  - Decision: Used `interface` for extendable object shapes (Project, Service, SiteMeta, SiteLinks, ProjectSection) and `type` for unions (Locale, ProjectStatus). Interfaces allow declaration merging if needed later; types are cleaner for unions.
  - Decision: `icon` on Service is `string | undefined`, not an enum. Slice 03 picks the icon library — the data layer should not couple to it.
  - Learning note: `LocalizedString` makes i18n a data concern, not a UI concern. Components never branch on locale themselves — they call `resolveLocale()` and get a plain string back.

- [x] Create `src/lib/data/locale.ts`
  - Files changed: `src/lib/data/locale.ts`
  - Decision: Direct fallback to `en` only — no intermediate chain. `resolveLocale({ en: "Hello", fr: "Bonjour" }, 'es')` returns `"Hello"`, not `"Bonjour"`. Simple, predictable, matches the COALESCE analogy in the spec.
  - Decision: Empty strings are treated as missing values. An empty `fr: ""` falls through to English. This prevents half-filled translations from surfacing as blank UI.

- [x] Create `src/lib/data/locale.test.ts`
  - Files changed: `src/lib/data/locale.test.ts`
  - 10 tests covering all locale permutations, empty string edge case, whitespace edge case, and constant validation.

- [x] Create `src/lib/data/projects.ts`
  - Files changed: `src/lib/data/projects.ts`
  - Decision: Helper functions co-located with data. They operate on nothing outside this file — splitting them would be artificial.

  - Decision: Used `readonly Project[]` type annotation instead of `as const`. `as const` would narrow every string to its literal type, making the array incompatible with the `Project` interface that expects `string`.

### Session Start
- **Time:** ~14:00
- **Slice spec:** docs/slices/slice-07-blog.md
- **Goal:** Build /blog listing + /blog/personal + /blog/[slug] with markdown rendering, SVG illustrations, GSAP morph animations, search, tag/date/language filters

### Work Done
- [x] Blog route structure: /blog, /blog/personal, /blog/[slug]
  - Files created: `src/routes/blog/+page.svelte`, `src/routes/blog/+page.ts`, `src/routes/blog/personal/+page.svelte`, `src/routes/blog/personal/+page.ts`, `src/routes/blog/[slug]/+page.svelte`, `src/routes/blog/[slug]/+page.ts`
  - Decision: Two content lanes — professional (/blog) and personal (/blog/personal) — each with its own accent color (#E07800 vs #FFB627)
  - Learning note: SvelteKit dynamic routes use `[slug]` folder convention; `+page.ts` exports `load` function that runs server-side

- [x] Blog data layer: frontmatter parser + markdown rendering with `marked`
  - Files modified: `src/lib/data/blog.ts`, `src/lib/data/types.ts`, `src/lib/data/index.ts`
  - Decision: Used `marked` library for markdown-to-HTML instead of mdsvex for content rendering. mdsvex is still in svelte.config.js for .md file processing, but the actual HTML rendering uses `marked` because mdsvex had issues with `<` characters in inline code/content
  - Learning note: `import.meta.glob` with `{ as: 'raw', eager: true }` loads markdown files at build time — no runtime FS access needed

- [x] Per-post folder structure for blog content
  - Files created: `src/content/blog/professional/why-i-left-orm-for-raw-sql/index.md`, `src/content/blog/professional/building-a-transit-pipeline/index.md`, `src/content/blog/professional/anime-data-viz-challenge/index.md`, `src/content/blog/professional/lorem-data-warehousing/index.md`, `src/content/blog/professional/lorem-etl-patterns/index.md`, `src/content/blog/personal/lorem-transit-future/index.md`, `src/content/blog/personal/lorem-space-exploration/index.md`, `src/content/blog/_template.md`
  - Decision: Each post lives in its own folder (`category/slug/index.md`) to allow co-located assets (images, etc.) in the future
  - Learning note: The `_template.md` file provides a copy-paste starting point for new posts with all frontmatter fields documented

- [x] Blog listing components: BlogListingPage, BlogRow, BlogFilterSidebar, BlogFilterMobile
  - Files created: `src/lib/components/BlogListingPage.svelte`, `src/lib/components/BlogRow.svelte`, `src/lib/components/BlogFilterSidebar.svelte`, `src/lib/components/BlogFilterMobile.svelte`
  - Decision: BlogListingPage is shared between /blog and /blog/personal — configured via props (heading, accent color, corner link). This avoids duplicating filter/search logic.

- [x] Blog detail components: BlogDetailHeader, BlogContent
  - Files created: `src/lib/components/BlogDetailHeader.svelte`, `src/lib/components/BlogContent.svelte`
  - Decision: Detail page uses max-w-2xl centered layout for readability

- [x] SVG illustrations with BlogSvgIcon component
  - Files created: `src/lib/components/BlogSvgIcon.svelte`
  - Decision: Each post can specify a custom SVG or fall back to a deterministic per-slug fallback. Fallback names are computed from slug hash to ensure consistent illustration per post.
  - Learning note: MorphSVGPlugin hover effect — on mouseenter, all SVG paths morph into a random geometric shape (triangle, circle, square, hexagon); on mouseleave they morph back. Never picks the same shape twice in a row.

- [x] Search, tag filters, date range filter, language filter
  - Files modified: `src/lib/components/BlogListingPage.svelte`
  - Decision: All filtering is client-side (small dataset, no need for server round-trips). Search matches title and excerpt. Date range uses native HTML date inputs. Language filter uses the `lang` field from frontmatter.

- [x] Updated Nav with /blog link
  - Files modified: `src/lib/components/Nav.svelte`
  - Decision: Blog link was already in Nav links array from Slice 06d — no structural change needed, just verified routing works

- [x] mdsvex configuration in svelte.config.js
  - Files modified: `svelte.config.js`
  - Decision: Added mdsvex preprocessor and `.md` extension so SvelteKit recognizes markdown files. Content rendering still uses `marked` for the actual HTML output.

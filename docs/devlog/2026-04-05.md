### Work Done

#### Fix 1: Scroll-linked text animations (Part 3b)
- Consulted GSAP MCP for `containerAnimation` pattern
- Root cause: `toggleActions` + `yoyo` + `repeat` don't sync with `containerAnimation` scrub; gradient CSS properties (`backgroundImage`, `backgroundClip`) aren't animatable by GSAP
- Fixed all 4 panels to use pure `scrub: true` — animations naturally reverse on scroll back
- Files changed: `src/lib/components/SkillsJourney.svelte`

#### Fix 2: Complete animation redesign (per Yesid's vision)
- Replaced all keyword animations with cinematic, meaning-driven effects:
  - **foundation** — structural assembly (chars rise from below and lock into position)
  - **data** — text scramble (chars cycle through `!<>-_\/#01234ABCDEF`, resolve one by one)
  - **logic** — precise assembly (chars spread from center, converge inward mathematically)
  - **pixels** — fragmented particles (tiny/rotated/blurred fragments snap to clean type)
  - **understand** — disorder → comprehension (scattered/blurred chars resolve in random order)
  - **unforgettable** — ghosted → permanent (barely visible chars reclaim solidity in two phases)
  - **stop** (CTA) — train braking (horizontal momentum + blur → crisp yellow lock-in)
- Kept: "Stations" wave + "motion" rotation (unchanged per Yesid's request)
- Decision: Used `trigger: hw` (per-word) instead of `trigger: el` (per-paragraph) for accurate timing
- Learning note: With `containerAnimation`, the trigger element determines WHEN the animation fires relative to the horizontal scroll. Using the paragraph as trigger means words at the end of the sentence animate off-screen. Using the actual word element fixes this.

#### Fix 3: Timing and trigger adjustments
- Changed all per-word ScrollTrigger triggers from `trigger: el` to `trigger: hw`
- Shifted all start/end values later so animations play when words are centered on screen
- Decision: `start: 'left 55%', end: 'left 15%'` as the baseline range for most effects

#### Fix 4: Remove non-keyword text animations
- Removed fade-in + y-slide on white (non-highlight) text per Yesid's feedback
- Only keywords animate now — base text is always static and readable
- Reduces GSAP tween count for lighter performance

#### Fix 5: "stop" brake animation fixes
- Made initial state subtle (x:10, scaleX:1.04, opacity:0.75, blur:0.5px) so word is always readable
- Added yellow (#FFB627) as final color
- Range: `start: 'left 75%', end: 'left 50%'` — completes by center so it works on mobile too

#### Fix 6: Panel 1 base text color
- Changed from grey #999 to white #f5f5f5 (same as all other panels) per Yesid's request

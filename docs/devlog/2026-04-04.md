## Slice: C — Zoom Transition

### Session Start
- **Time:** 19:30
- **Slice spec:** docs/slices/slice-c-zoom-transition.md
- **Goal:** Transition fully orange hero screen into brand statement via zoom-out

### Work Done

- [x] Hero text markup added to HeroBanner.svelte
  - Files changed: `src/lib/components/HeroBanner.svelte`, `src/routes/home.test.ts`
  - Decision: Split layout (headline left, SQL decoration right) — chosen over centered or left-aligned
  - Decision: The "." dot after INFRA is a sibling span, NOT nested inside a `data-hero-stagger` element — so it stays visible during cross-fade while other text is hidden
  - Learning note: The hero text container starts scaled up massively (transform-origin at the dot). As it scales down to 1, text "appears" around the dot.

- [x] Extended GSAP timeline with cross-fade + zoom-out
  - Files changed: `src/lib/components/HeroBanner.svelte`
  - Decision: Added phases at timeline position 1.0→1.5, scroll distance 800% → 1200%. No remapping of existing Slice A phases — GSAP scrub maps proportionally.
  - Decision: Cross-fade at position 1.0 (SVG opacity→0, hero text container opacity→1). Zoom-out at 1.05→1.40.
  - Learning note: Adding tweens after timeline position 1.0 extends totalDuration. With scrub, scroll 0-100% maps to timeline 0-totalDuration. So existing phases keep exact same pacing.

- [x] Text stagger reveal during zoom-out
  - Decision: Stagger order: dot (always visible) → DATA/INFRA → BUILT RIGHT. → badge/subtitle/CTAs → SQL decoration
  - Decision: Scoped `querySelectorAll('[data-hero-stagger]')` to heroTextContainer to avoid targeting other DOM elements

- [x] Dot glyph centering — DOM baseline probe + Canvas metrics
  - Decision: Used zero-height `inline-block` probe at `vertical-align:baseline` to find the actual baseline in browser layout. Canvas `measureText('.')` provides `actualBoundingBoxAscent/Descent` for the glyph offset. This is the only reliable cross-font way to center the zoom on the visible "." glyph (not its layout box).
  - Learning note: `getBoundingClientRect()` on a text span returns the line box (includes line-height), NOT the visible glyph. At 100x+ scale, even a few pixels of offset between transform-origin and glyph center gets magnified enormously.

- [x] Background transparency
  - Decision: Animate pinContainer background to transparent during zoom-out so page gradient shows through

- [x] Resize handling + reduced motion
  - Decision: Same getBoundingClientRect pattern as Slice A zoom-in, extended with hero text origin + scale recalculation
  - Decision: Reduced motion shows hero text statically at scale(1) opacity(1)

### Work Done

- [x] Create `use:tilt` action (`src/lib/motion/actions/tilt.ts`)
  - Files changed: `tilt.ts`, `tilt.test.ts`, `actions/index.ts`
  - Decision: Copied `isTouchDevice()` inline from magnetic.ts (matches existing pattern, no shared utility extraction needed)
  - Learning note: The tilt action follows the same pattern as magnetic — cursor offset mapped to CSS transform via perspective + rotateX/rotateY. The key insight is that rotateX controls vertical tilt (inverted) and rotateY controls horizontal tilt.

- [x] Extend LottiePlayer with scrub/progress props
  - Files changed: `LottiePlayer.svelte`, `setup.ts`
  - Decision: Added `scrub` and `progress` as optional props with defaults (false, 0). When scrub=false, all existing behavior is preserved byte-for-byte. The `DOMLoaded` event captures totalFrames safely.
  - Learning note: `animation.goToAndStop(frame, true)` — the second arg `true` means "frame number" not "time in seconds". This is how you seek to a specific frame.

- [x] Rewrite train-path.ts for vertical trajectory
  - Files changed: `train-path.ts`, `train-path.test.ts`
  - Decision: Same cubic bezier structure (M, C, S), just swapped axes. Station waypoints flatten to baseX (no wobble) for clean alignment with rail nodes. Control point offsets on Y axis.
  - Learning note: The path format is pure math — GSAP MotionPathPlugin doesn't care about direction. The SVG path string works identically for horizontal or vertical travel.

- [x] Update TrainJourney for vertical orientation + velocity glow + easter egg
  - Files changed: `TrainJourney.svelte`
  - Decision: CSS `rotate(90deg)` on inner wrapper turns nose downward without touching Train.svelte markup. Velocity glow via CSS custom property `--train-glow` modulating a `drop-shadow` filter. Easter egg is a simple GSAP bounce on click.
  - Learning note: Separating the rotation from the motion path means GSAP handles position and CSS handles orientation independently.

- [x] Rewrite +page.svelte layout
  - Files changed: `+page.svelte`
  - Decision: Removed Threlte 3D scene entirely (dynamic import + rendering). Replaced with universal CSS gradient + scroll-reactive warm glow overlay. Right-rail visual (crossties, station nodes, dashed connectors) rendered inline in the page's fixed overlay layer, not in ScrollRail. ScrollRail untouched.
  - Learning note: The rail overlay uses CSS repeating gradients for crossties and absolute-positioned divs for station nodes. Station node positions are data-driven: `15 + (i / (services.length - 1)) * 70` percent from top.

- [x] Restyle ServiceStation as station sign card
  - Files changed: `ServiceStation.svelte`, `ServiceStation.test.ts`
  - Decision: Station number derived from `index + 1` (zero-padded). Indicator light active state via IntersectionObserver with `rootMargin: '-30% 0px'`. Lottie in scrub mode (400x400, scroll-linked). Card tilt via `use:tilt`. Added `use:reveal` for scroll entrance.
